# 产品实习文件夹结构说明

本文件夹采用PARA方法变体进行组织，用于系统化管理产品实习相关的知识和材料。

## 文件夹结构说明

### 00_Inbox

**所有临时记录先丢这里。**

比如：
- 课程里听到的新概念
- 看文章时的临时摘录
- 面试题灵感
- 项目中的零散想法

**这里的内容不要求完整，只要求先记下来。**

### 01_Concepts

**放"概念卡"。**

比如：
- 北极星指标
- AARRR
- 用户画像
- RAG
- Agent
- 留存率
- 转化漏斗

**标准：一篇笔记只讲清一个概念。**

### 02_Methods

**放"方法卡"。**

比如：
- 用户访谈怎么做
- PRD 怎么写
- 竞品分析怎么做
- 需求优先级排序
- A/B Test 设计方法
- 指标拆解方法

**标准：一篇笔记只讲清一个做事方法。**

### 03_Cases

**放"案例拆解"。**

比如：
- ChatGPT 的产品路径拆解
- 某 AI 工具的 onboarding 分析
- 某产品的增长机制拆解
- 某次失败功能上线复盘

**标准：聚焦一个案例，一个问题，一组洞察。**

### 04_Projects

**放你自己的实战项目。**

比如：
- AI 简历优化产品练习
- 校园二手平台需求分析
- 某次面试作业
- 某次实习中的功能方案

**这里不放最终交付文档正文，放的是：**
- 项目理解
- 思考过程
- 决策记录
- 复盘
- 关联知识

### 05_MOCs

**MOC = Map of Content，内容地图。**

它不是知识正文，而是导航页。

比如：
- AI产品经理学习地图
- 用户研究知识地图
- 数据分析知识地图
- 我的项目总览
- 求职准备导航页

### 06_Resources

**放参考资料索引，不放"你自己的理解正文"。**

比如：
- 文章链接汇总
- 课程目录
- 书单
- 播客清单
- 招聘 JD 收集

**这里更像"资料入口"。**

### 07_Templates

**放模板。**

比如：
- 概念笔记模板
- 方法笔记模板
- 项目笔记模板
- 案例拆解模板
- 周复盘模板

### 99_Archive

**过期内容归档。**

比如：
- 已失效的学习计划
- 重复笔记
- 老版本项目记录
- 不再维护的材料

## 使用建议

1. **定期整理**：每周整理一次00_Inbox，将内容移动到合适的文件夹
2. **保持简洁**：每个文件聚焦一个主题，避免内容混杂
3. **建立链接**：使用Obsidian的链接功能连接相关笔记
4. **持续更新**：随着学习深入，不断更新和完善内容
5. **定期归档**：每季度整理一次，将不再活跃的内容移动到99_Archive
