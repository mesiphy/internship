# 安装所需的环境依赖

> 学习编程过程中常见环境/工具的分类、用途与安装方式总览。

---

## 一、分类总览表

| 分类 | 工具名称 | 功能简介 | 安装对象 | Windows 安装方式 | Mac 安装方式 | 检测命令 |
|---|---|---|---|---|---|---|
| **OS 级包管理器** | **Homebrew** | macOS 系统级包管理器，一键安装各类底层软件 | Git、Node.js、Python、uv 等 | ❌ 不支持（可用 winget 替代） | `/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"` | `brew --version` |
| **OS 级包管理器** | **winget** | Windows 系统级包管理器（Win10/11 内置） | Git、Node.js、Python、VS Code 等 | 系统自带，终端输入 `winget` 即可 | ❌ 不支持 | `winget --version` |
| **版本管理器** | **nvm** | 管理/切换多个 Node.js 版本 | Node.js | 下载 [nvm-windows](https://github.com/coreybutler/nvm-windows/releases) 安装包（路径禁含空格/中文） | `brew install nvm`，需手动配 `~/.zshrc` | `nvm --version` |
| **版本管理器** | **pyenv** | 管理/切换多个 Python 版本 | Python | `pyenv-win`（GitHub Releases 下载） | `brew install pyenv` | `pyenv --version` |
| **语言运行时** | **Node.js** | JavaScript 运行环境（脱离浏览器），前后端基础底座 | — | 推荐用 nvm 安装：`nvm install lts` → `nvm use lts` | 推荐：`brew install node` 或 `nvm install lts` | `node --version` |
| **语言运行时** | **Python** | Python 解释器，AI/数据分析/后端开发基础 | — | `winget install Python.Python.3.12` 或官网 `.exe` | `brew install python@3.12` | `python3 --version` |
| **语言运行时** | **Java (JDK)** | Java 开发工具包，企业级后端/安卓开发基础 | — | `winget install Oracle.JDK.21` 或官网下载 | `brew install openjdk@21` | `java --version` |
| **语言运行时** | **Go** | Go 语言运行时，云原生/高并发后端 | — | `winget install GoLang.Go` | `brew install go` | `go version` |
| **语言运行时** | **Rust** | Rust 语言工具链（含编译器 + Cargo） | — | 官网下载 `rustup-init.exe` | `curl --proto '=https' --tlsv1.2 -sSf https://sh.rustup.rs | sh` | `rustc --version` |
| **语言级包管理器** | **npm** | Node.js 包管理器，世界最大软件注册表 | JS 第三方库（React、Vue 等） | 随 Node.js 捆绑安装 | 随 Node.js 捆绑安装 | `npm --version` |
| **语言级包管理器** | **yarn** | Node.js 包管理器（替代 npm，更快更稳定） | JS 第三方库 | `npm install -g yarn` | `npm install -g yarn` | `yarn --version` |
| **语言级包管理器** | **pnpm** | Node.js 包管理器（磁盘高效、严格） | JS 第三方库 | `npm install -g pnpm` | `npm install -g pnpm` | `pnpm --version` |
| **语言级包管理器** | **uv** | 极速 Python 包管理器（Rust 编写，替代 pip） | Python 第三方库（Numpy、Requests 等） | `powershell -c "irm https://astral.sh/uv/install.ps1 | iex"` | `brew install uv` 或 `curl -LsSf https://astral.sh/uv/install.sh | sh` | `uv --version` |
| **语言级包管理器** | **pip** | Python 传统包管理器 | Python 第三方库 | 随 Python 捆绑安装 | 随 Python 捆绑安装 | `pip --version` |
| **语言级包管理器** | **Cargo** | Rust 包管理器 + 构建工具 | Rust 第三方库 | 随 Rust（rustup）一起安装 | 随 Rust（rustup）一起安装 | `cargo --version` |
| **版本控制** | **Git** | 分布式版本控制系统，代码历史记录与团队协作 | — | `winget install Git.Git` 或官网下载（带 Git Bash） | `brew install git` 或终端输入 `git` 按提示安装 Xcode CLI | `git --version` |
| **容器/虚拟化** | **Docker** | 容器化平台，将应用及其依赖打包成隔离环境运行 | — | 下载 Docker Desktop for Windows | 下载 Docker Desktop for Mac 或 `brew install --cask docker` | `docker --version` |
| **终端/Shell** | **Git Bash** | Windows 下的类 Unix 终端（随 Git 安装） | — | 随 Git for Windows 自动安装 | ❌ 不需要（Mac 自带终端） | — |
| **终端/Shell** | **zsh** | 增强型 Shell（Mac 默认），支持插件和主题 | — | ❌ 不支持 | Mac 自带（Catalina+），配合 Oh My Zsh | `zsh --version` |
| **终端/Shell** | **PowerShell** | Windows 现代 Shell（比 CMD 强大） | — | Win10/11 自带 | `brew install powershell` | `pwsh --version` |
| **终端/Shell** | **Windows Terminal** | 微软官方现代终端模拟器（支持多标签） | — | 微软商店下载或 `winget install Microsoft.WindowsTerminal` | ❌ 不支持 | — |
| **编辑器** | **VS Code** | 最流行的免费代码编辑器，丰富插件生态 | — | `winget install Microsoft.VisualStudioCode` | `brew install --cask visual-studio-code` | `code --version` |

## 二、包管理器的层级关系（核心理解）

这是配置开发环境时最容易混淆的地方。不同管理器的"管辖范围"不同：

| 管理器名称 | 管理层级 | 管辖范围 | 典型安装命令示例 | 形象比喻 |
|---|---|---|---|---|
| **Homebrew**（Mac）/ **winget**（Win） | **操作系统级** | 安装系统基础软件和语言环境（如 Git、Node.js、Python 本身） | `brew install node` / `winget install OpenJS.NodeJS` | **建材市场**：帮你买来"厨房"（Node.js）和"书房"（Python） |
| **nvm** / **pyenv** | **版本控制级** | 专门负责某一种语言的**多版本安装与切换** | `nvm install 18` / `pyenv install 3.12.0` | **装修工**：把厨房随时改造成中式或西式（切换 Node 版本） |
| **npm** / **yarn** / **pnpm** | **语言/项目级** | 依附于 Node.js，下载 JS 写的第三方代码包 | `npm install react` | **厨具店**：在厨房里买锅碗瓢盆 |
| **uv** / **pip** | **语言/项目级** | 依附于 Python，下载 Python 写的第三方代码包 | `uv pip install numpy` / `pip install requests` | **文具店**：在书房里买笔和纸 |

```

                    │   OS 级包管理器        │
                    │  Homebrew / winget    │
                    │  "建材市场"            │
                    └──────┬───────────────┘
                           │ 安装基础软件
              ┌────────────┼────────────┐
              ▼            ▼            ▼
        ┌──────────┐ ┌──────────┐ ┌──────────┐
        │  Git     │ │ Node.js  │ │ Python   │
        │          │ │          │ │          │
        └──────────┘ └────┬─────┘ └────┬─────┘
                          │            │
                    ┌─────▼─────┐ ┌───▼───────┐
                    │   nvm     │ │  pyenv    │  ← 版本管理器
                    │ "装修工"   │ │ "装修工"   │
                    └─────┬─────┘ └───┬───────┘
                          │           │
              ┌───────────┼───┐ ┌─────┼──────────┐
              ▼           ▼   ▼ ▼     ▼          ▼
        ┌─────────┐ ┌─────────┐ ┌─────────┐ ┌─────────┐
        │  npm    │ │  yarn   │ │   uv    │ │   pip   │ ← 语言级包管理器

        └─────────┘ └─────────┘ └─────────┘ └─────────┘
```

## 三、典型安装顺序（新手建议）

### Mac

```bash
# 第 1 步：安装 OS 级包管理器（先有"建材市场"）
/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"

# 第 2 步：安装版本管理器 + Git
brew install git nvm pyenv

# 第 3 步：用版本管理器安装语言运行时
nvm install --lts          # Node.js 长期支持版
pyenv install 3.12.0       # Python 指定版本

# 第 4 步：语言级包管理器随语言自带，直接可用
npm --version              # npm 随 Node.js 已就绪
pip --version              # pip 随 Python 已就绪
brew install uv            # uv 推荐单独安装
```

### Windows

```powershell
# 第 1 步：winget 系统自带，无需安装

# 第 2 步：安装 Git + 版本管理器
winget install Git.Git
# nvm-windows：去 GitHub Releases 下载安装包

# 第 3 步：用版本管理器安装语言运行时
nvm install lts
nvm use lts

# 第 4 步：语言级包管理器随语言自带
npm --version
```

## 四、你可能遇到的问题

### sudo 与权限问题

在 Mac/Linux 下，`sudo` 以超级管理员（root）身份执行命令。安装软件时可能遇到 `Permission denied`。

- **原因**：试图往系统目录写文件但没有权限。
- **解决**：尽量使用 Homebrew 安装软件（它会安装到用户目录，不要求 sudo）。避免用 `sudo npm install -g`，而应使用 nvm 管理 Node 版本。
- **注意**：误用 `sudo` 可能导致文件所有权混乱（root 拥有你的个人文件），后续无法正常修改。

### 环境变量

在配置各种 Code Agent 和开发工具时，**环境变量**是关键：

- **PATH**：告诉系统去哪些目录找可执行文件。安装新工具后，如果终端提示 "command not found"，通常是 PATH 没配好。
- **常见需要配 PATH 的场景**：nvm、pyenv、uv、Java JDK、Go 等。
- **如何查看当前 PATH**：
  - Mac/Linux：`echo $PATH`
  - Windows PowerShell：`$env:PATH`

## 五、快速自查清单

新装系统后，确认以下命令都有输出即为就绪：

| 检查项 | 命令 | 说明 |
|---|---|---|
| Git | `git --version` | 版本控制 |
| Node.js | `node --version` | JS 运行时 |
| npm | `npm --version` | JS 包管理器 |
| Python | `python3 --version` | Python 运行时 |
| pip | `pip --version` | Python 包管理器 |
| uv | `uv --version` | 现代 Python 包管理器 |
| Homebrew（Mac） | `brew --version` | 系统包管理器 |
| VS Code | `code --version` | 编辑器 |
| Docker | `docker --version` | 容器（按需） |
