## 定义
任务完成率 是 Skills / Workflow / Agent 中用于判断效果、质量或系统状态的关键指标，关注的是 任务编排、智能体执行与人工协同。

## 核心问题
核心问题是：任务完成率 到底在衡量什么、阈值如何设定、异常时说明哪一环出了问题。

## 应用场景
- 把多步任务拆成可执行的工作流或 Agent。
- 设计 planner、executor、human review 和 handoff。
- 控制自动执行边界、终止条件和成功率指标。

## 常见误区
- 把 Agent 理解成“自动多轮调用”而忽略状态与兜底。
- 在高风险动作上过度自动化。
