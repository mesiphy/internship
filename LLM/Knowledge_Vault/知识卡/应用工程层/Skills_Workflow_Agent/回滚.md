## 定义
回滚 是 Skills / Workflow / Agent 中的观测与定位抓手，用于连接运行现象、版本变化和问题归因。

## 核心问题
核心问题是：如何借助 回滚 观察系统运行状态，并把现象准确归因到模型、检索、工具或流程。

## 应用场景
- 把多步任务拆成可执行的工作流或 Agent。
- 设计 planner、executor、human review 和 handoff。
- 控制自动执行边界、终止条件和成功率指标。

## 常见误区
- 把 Agent 理解成“自动多轮调用”而忽略状态与兜底。
- 在高风险动作上过度自动化。
