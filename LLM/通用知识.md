这个视频从最底层的工程视角出发，由浅入深地拆解了AI领域的核心概念，帮助观众打通底层逻辑。以下是视频各个章节的具体内容总结：

**1. 大语言模型 (LLM)**

- **概念与背景**：LLM全称是大语言模型，目前的大模型基本都是基于Google在2017年提出的Transformer架构训练出来的，随后由OpenAI通过GPT系列将其引爆全世界。
- **工作原理**：大模型的工作原理本质上是一个“**文字接龙游戏**”。它接收输入后，会预测下一个概率最高的词，然后将这个词追加到原有输入后面继续预测，直到输出一个特殊的结束标识符为止，因此大模型总是逐词输出答案。

**2. Token 与 Tokenizer**

- **中间人 Tokenizer**：大模型本质上是数学函数，只认识数字不认识文字，因此需要Tokenizer作为“翻译官”。它负责**编码**（将文字切分为最小片段Token，并映射为数字Token ID）和**解码**（将模型生成的Token ID还原成文字）。
- **Token 的切分规则**：Token才是大模型处理文本的基本单元。Token和词并不是一一对应的关系，它是一套模型自己学会的文本切分规则。平均而言，一个Token大约等于0.75个英文单词或1.5到2个汉字。

**3. 上下文 (Context) 与 上下文窗口 (Context Window)**

- **上下文 (Context)**：大模型本身没有记忆，它之所以能“记住”之前的聊天，是因为背后的程序每次都会将整个对话历史和用户问题一起发送给模型。Context就是大模型每次处理任务时接收到的信息总和（包括历史记录、正在输出的Token、系统规则等），相当于大模型的**临时记忆体**。
- **上下文窗口 (Context Window) 与 RAG**：代表Context能容纳的**最大Token数量**。如果遇到超长文本（如千页产品手册），把全部内容丢给大模型成本很高且容易超出限制，这时可以借助**RAG技术**，只抽取与用户问题最匹配的几个片段发给模型来回答。
[[RAG]]

**4. 提示词 (Prompt)**

- **提示词的定义**：提示词是给大模型的具体问题或指令，写得越清晰、具体、明确，模型输出的质量就越高。
- **两种 Prompt**：
    - **User Prompt (用户提示词)**：用户自己在对话框中输入的具体任务要求。
    - **System Prompt (系统提示词)**：开发者在后台配置的，用户看不见，但用于**设定大模型的人设和做事规则**（例如要求一个数学辅导机器人只能引导思考，不能直接给答案）。

**5. 工具 (Tool) 与 MCP协议**

- **工具 (Tool)**：大模型无法感知外界环境（如查询实时天气），Tool本质上就是一个**函数**，用来给大模型提供外部能力。流程上，模型会根据需求向平台输出“调用工具”的指令，平台执行调用后将结果返回给模型，模型最后再将结果归纳总结输出给用户。
- **MCP (模型上下文协议)**：由于每个平台（如OpenAI, Anthropic, Google）接入工具的规范不同，开发者需要写多套代码。MCP（Model Context Protocol）应运而生，它是一套**统一的工具接入标准**（类似手机的Type-C接口），开发者只需按此规范写一次代码，工具就能在所有支持MCP的平台上使用。
- [[MCP]]

**6. 智能体 (Agent)**

- 当面对需要多次调用不同工具（如先查定位、再查天气、最后查附近便利店）的复杂任务时，就需要Agent登场。
- **Agent的定义**：Agent是一个**能够自主规划、自主调用工具、一步步思考并持续运作**，直到最终完成用户任务的系统。
[[Agent]]

**7. Agent Skill**

- **用户痛点**：在实际高频使用中，用户可能有特定的私人习惯或输出格式要求（例如出门提醒要求先总结、再列清单），每次提问都附带一大串规则非常繁琐。
- **Agent Skill的作用与结构**：它本质上是一份**提前写好给Agent看的Markdown说明文档**，用来规定做事步骤和规则。包含**元数据层**（名称和描述）和**指令层**（目标、执行步骤、判断规则、输出格式和示例）。
- **运行机制**：Agent Skill存放在指定目录下，平台启动时只加载元数据。只有当用户问题与该技能名称和描述相关时，才会读取具体的指令层，这种**渐进式披露机制**可以节省大量Token。
